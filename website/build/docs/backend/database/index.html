Database Schema

The project uses PostgreSQL as the primary data store. Below is the schema design for the core modules.

Core Tables​

users​

Stores authentication and profile information.

  • id (UUID/Serial): Primary Key
  • email (String): Unique
  • password_hash (String)
  • username (String): Unique
  • created_at (Timestamp)

rituals​

Stores the habits created by users.

  • id (Serial): Primary Key
  • user_id (FK -> users.id)
  • title (String)
  • description (Text)
  • frequency (Enum): 'daily', 'weekly', etc.
  • reminder_time (Time)
  • is_archived (Boolean)

ritual_logs​

Records every completion of a ritual.

  • id (Serial): Primary Key
  • ritual_id (FK -> rituals.id)
  • completed_at (Timestamp)
  • note (Text): Optional user note

Gamification Tables​

user_profiles​

Stores gamification stats.

  • user_id (FK -> users.id)
  • xp (Integer): Total experience points
  • level (Integer): Current level
  • coins (Integer): Virtual currency
  • current_streak (Integer)
  • longest_streak (Integer)

badges​

Definitions of available badges.

  • id (Serial)
  • name (String)
  • description (String)
  • icon_url (String)
  • xp_reward (Integer)

user_badges​

Badges earned by users.

  • user_id (FK)
  • badge_id (FK)
  • earned_at (Timestamp)

Social Tables​

friendships​

Manages friend connections.

  • requester_id (FK)
  • addressee_id (FK)
  • status (Enum): 'pending', 'accepted', 'rejected'

partnerships​

Manages 1v1 ritual partnerships.

  • id (Serial)
  • ritual_id (FK)
  • user1_id (FK)
  • user2_id (FK)
  • status (Enum): 'active', 'ended'
  • partnership_streak (Integer)
